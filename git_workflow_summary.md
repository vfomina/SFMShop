# Git Workflow для проекта SFMShop

## Использованные команды Git
- git init - инициализация репозитория
- git add - добавление файлов в staging area
- git switch <ветка> / git checkout <ветка> - переключение на ветку
- git switch -c <ветка> - создать и переключиться на ветку
- git merge <ветка> - слияние веток
- git reset --soft HEAD~1 - убрать последний локальный коммит без удаления изменений
- git push -u origin <ветка> - загрузка изменений в репо, первая публикация
- git log - просмотр истории коммитов
- git branch - просмотр всех веток
- git remote -v - просмотр удаленных репозиториев
- git pull - получение истории с удаленного репозитория
- git log --stat - просмотр подробной статистики с изменениями файлов
- git rebase - перебазирование на основе другой ветки
- git commit - добавить коммит с сообщением

## Созданные ветки
- main - основная ветка проекта
- add-discount-system - ветка для системы скидок
- feature/add-email-validation - ветка для системы валидации email
- feature/add-inventory-management - ветка для управления складом
- feature/add-shipping - ветка для управления доставкой
- feature/test-conflict - ветка для учебного моделирования конфликта слияния

- Конфликт в src/models/product.py при слиянии feature/add-discount-system
- Конфликт в src/models/product.py при слиянии feature/add-shipping
## Стратегия работы с ветками
1. Создавать отдельную ветку с осмысленным названием для работы с отдельными линиями разработки (фича, фикс)
2. Делать PR в основную ветку, с добавлением описания (что изменилось, зачем, как проверить) и делая squash, если коммитов слишком много
3. В рамках одной ветки не делать смыслово другие изменения, не относящиеся к ней
4.  Перед слянием личной ветки в основную делать rebase (либо merge)
5. Добавлять осмысленные сообщения коммитов
6. Удалять ветки, если уже не будут использоваться
